题目描述：输入n，从1打印到最大的第n位数。

思路：题目没有说明n的范围，有可能超出编程语言自带整数数据结构的范围，应该按照大数情况处理。按照大数情况处理。

把打印数字转换成数字排列的解法，递归让代码更简洁。在数字前面补0，n位所有十进制数就是n个从0到9的全排列。

18 在O(1)时间删除链表节点

题目描述：在O(1)的时间删除节点。

这是leetcode的题解，时间复杂度是O(n)
leetcode link

按照O(1)的题目要求做。如果要删除的节点是尾巴节点，就只有从头扫到尾来删除。如果不是尾巴节点，把下一个节点的值复制到当前要删除的节点中，再删除下一个节点。这个方法基于一个假设，就是假设将要删除的节点就在链表当中。如果受O(1)时间的限制，就需要把确保节点在链表中的责任交给函数的调用者。

19 正则表达式匹配 （难）

题目描述：
请实现一个函数用来匹配包含‘. ’和‘*’的正则表达式。模式中的字符‘.’表示任意一个字符，而‘*’表示它前面的字符可以出现任意次（含0次）。

解题思路：动态规划

  • 定义状态：dp[i][j]表示s(string)的前i个字符和p(pattern)的前j个字符是否匹配。
  • 转移方程：
    • 如果s[i]==p[j]，或 p[j]==’.’，$dp[i][j]=dp[i-1][j-1]$
    • 如果p[j]==’*’:
      • 正则串表示 0 个字符，直接砍掉正则串的后面两个,$dp[i][j]=dp[i][j-2]$
      • 正则串表示当前字符出现多次，正则串不动，主串向前移动一位，$d p[i][j]=d p[i-1][j]$

详细思路：
首先关注正则表达式P的最后一个字符，有三种可能，正常字符、 星号\*、点 .。针对这3种情况讨论就可以了。

  1. 模式Pattern的最后一个字符是正常字符，查看 S[n-1]是否等于P[m-1]，相等则查看子问题$S_{0..n-2}$和$P_{0..m-2}$是否匹配。
  2. 如果Pattern的最后一个字符是点’.’，直接查看$S_{0..n-2}$和$P_{0..m-2}$是否匹配。
  3. 如果Pattern的最后一个字符是星号’*’，这个代表$P[m-2]=c$重复0次或很多次。它们为一个整体。
    3.1 $S[n-1]$是0个c，忽略Pattern最后两个字符。则查看$S[n-1]$和$P[m-3]$是否匹配
    3.2 $S[n-1]$是多个c的最后一个（$S[n-1]==’c’$或者$P[m-2]=c=’.’$）。A匹配完往前挪动一个，P继续匹配。应为可以匹配很多个，继续看$A_{0..m-2}$和$B_{0..m-1}$是否匹配。

转移方程：
1.、2.：前两种情况合并成同一种$f[i][j]=f[i-1][j-1]$

  1. 分为看c和不看c：
    3.1 不看，直接砍掉正则串后面两个，$f[i][j]=f[i][j-2]$
    3.2 看，正则串不动，主串前移一个，$f[i][j]=f[i-1][j]$

初始状态：
注意：空串和空正则

  • 空串和空正则是匹配的 $f[0][0]=true$;
  • 空串和非空正则，必须要计算出来，不是星号’*‘的非空正则就是错的
  • 非空串和空正则必不匹配，$f[1][0]=\ldots=f[n][0]=\text {false}$

我们开数组要开 $n+1$ ，这样对于空串的处理十分方便。结果就是 $f[n][m]$
