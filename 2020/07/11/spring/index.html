Spring 的 AOP

为不具备继承依赖关系的对象引入一个公共行为。例如日志，权限，事务的功能。把这些行为放在业务代码里面不利于维护，希望不把这些工作放在各个处理过程中，不然不利于维护，也会有大量重复大码。

Spring会使用代理模式。

代理模式

Spring IOC

控制反转：控制对象生命周期的不再是引用它的对象，而是由容器来负责。把对象创建，销毁的控制能力由引用它的对象反转给容器。用容器来控制所有对象的生命周期和对象间的关系。

一个对象需要依赖另一个对象时，只需要告诉spring，系统动态提供它所需要的其他对象，这叫依赖注入。Spring通过反射实现依赖注入。

一句话就是，获得依赖对象的方式反转了。

Bean就是IOC的容器，将TestBeanService注册到Service里面，那么TestBeanService就是Spring里面的一个Bean。Demo里面context.getBean()就是从Spring中取出这个Bean，完成控制反转。

我们重点就是看Spring怎么生成管理这些Bean的。

ClassPathXmlApplicationContext 完成控制反转。

  • ClassPathXmlApplicationContext类是AbstractApplicationContext抽象类的子类
  • AbstractApplicationContext类是ApplicaionContext接口的实现。
  • ApplicaionContext接口集合了非常多的内容，其中和IOC比较相关的就是ListableBeanFactory接口和HierarchicalBeanFactory接口
  • ListableBeanFactory接口和HierarchicalBeanFactory接口是继承BeanFactory

ApplicationContext and BeanFactory

ApplicationContext是整个项目的配置，Spring项目启动或运行的时候都需要依赖它。

和Bean管理相关的是ListableBeanFactory和HierarchicalBeanFactory

BeanFactory就是获取Bean容器的地方，而且他可以提供单例的对象或者是独立的对象。

以上是基本的概念和认知，带着这些概念回头看ClassPathXmlApplicationContext执行流程，看看它怎么生成管理Bean的。

这个接口是要实现预先加载Bean的配置，生成好实例，直接管理Bean的实例，而不是来一个请求，生成一个。

1 ApplicationContext是Spring项目的核心配置，项目运行依赖于它，其中包含许多方面的内容。

3 Spring控制反转为了降低项目耦合，提高延展性。

Maven

maven是项目管理工具，
pom.xml的pom就是项目对象模型的意思
pom定义项目的坐标、项目依赖、项目信息、插件目标、打包方式等

  • maven优缺点：
  1. 简化了项目构建，依赖管理
  2. 方便容易上手
  3. 便于项目升级，无论是项目本身还是项目使用的依赖
  4. 有助于多么快项目开发。一个模块开发好后，发布到仓库，依赖该模块时可以直接从仓库更新，而不用自己去编译。
  5. maven有很多插件，便于功能扩展，比如生产站点，自动发布版本。

缺点：
庞大的构建系统，学习难度大
“约定优于配置”的策略，上手容易，出问题很难调适

  • 坐标

[groupId, artifactId, version, packaging, classifier]表示某个版本，有时还会使用classifier来表示项目的附属构建，常见的附属构建有javadoc和sources包。前3个必须定义，packaging可选（默认jar），classifier不能直接定义，需要结合插件。

  • groupId：项目组 Id，必须全球唯一；
  • artifactId：项目 Id，即项目名；
  • version：项目版本；
  • packaging：项目打包方式。

仓库

  • 本地仓库用来存储项目的依赖库；
  • 中央仓库是下载依赖库的默认位置；
  • 远程仓库，因为并非所有的依赖库都在中央仓库，或者中央仓库访问速度很慢，远程仓库是中央仓库的补充。

  1. clean周期：真正的构建之前，进行一些清理工作

  2. 默认周期：核心部分，编译、测试、打包、部署

  3. site周期 生成项目报告，站点、发布站点

SpringBoot

简化了使用Springboot的难度，省去了繁重的配置，提供各种启动器

  • 优点：
    使用JavaConfig避免使用xml
    避免大量的Maven导入和各种版本冲突
    提供意见发展方法
    通过提供默认值快速开始开发
    没有单独的Web服务器需要，不再需要启动Tomcat，Glassfish或其他任何东西。以Jar包的方式内嵌一个Tomcat。

SpringBoot自动装配原理

Spring Boot 本身就 Spring MVC 的简化版本。是在 Spring MVC 的基础上实现了自动配置，简化了开发人员开发过程。

Spring MVC 是通过一个叫 DispatcherServlet 前端控制器的来拦截请求的。而在 Spring Boot 中 使用自动配置把 DispatcherServlet 前端控制器自动配置到框架中。
